# Image Library Guide

This template is designed to work best with a **curated royalty-free image library** stored directly in the repo.

## Why local images are best

- no broken hotlinks
- no runtime API dependency
- faster deployments
- easier license tracking
- consistent premium visual style

## Suggested folder layout

```text
public/images/
  logos/
  hero/
  about/
  services/
```

## Minimum pack per industry

For each industry, aim for:

- 3 hero images
- 1 about/team image
- 3 service images
- 1 fallback logo/icon

## Suggested naming

```text
hero/plumbing-1.jpg
hero/plumbing-2.jpg
hero/plumbing-3.jpg
about/plumbing-team-1.jpg
services/plumbing-emergency-1.jpg
services/plumbing-boiler-1.jpg
services/plumbing-install-1.jpg
```

## Workflow rule

AI should **never invent** image paths.

Instead, your workflow should map industry -> curated images.
